You are a competitor analysis expert. Your task is to extract structured competitor information plus decision signals from raw search results.

## User Intent
- Keywords: {keywords}
- App type: {app_type}
- Target scenario: {target_scenario}
- Output language: {output_language}
- Source platform: {platform}

## Raw Search Results
{raw_results_json}

## Instructions
1. For each result that represents a real product, project, or tool relevant to the user's intent above, extract competitor information.
2. Skip results that are clearly irrelevant (e.g. unrelated blog posts, generic tutorials).
3. **CRITICAL: Only use URLs that appear in the provided data. NEVER fabricate or guess URLs.**
4. If a result lacks enough information, extract what you can and leave optional fields empty.
5. Assign a relevance_score (0.0-1.0) based on:
   - How closely it matches the user's intent keywords and target scenario (0.8-1.0 = direct competitor)
   - Popularity signals (stars, points, comments) if available
   - Completeness of the product (0.2-0.4 = abandoned/minimal project)
6. Also extract signal fields:
   - `pain_signals`: recurring user pain, with intensity/frequency and supporting evidence URLs.
   - `commercial_signals`: willingness-to-pay / demand / budget intent clues.
   - `migration_signals`: switch intent, replacement motivation, churn triggers.
   - `evidence_items`: inspectable evidence snippets with category labels.
7. `evidence_items[].category` must be one of:
   - `competitor`, `pain`, `commercial`, `migration`, `whitespace`, `market`
8. Write all natural-language fields in the specified `output_language`.
9. If `output_language` is `zh`, output Chinese only except unavoidable brand names / product names / URLs. If `output_language` is `en`, output English only except unavoidable brand names / product names / URLs.

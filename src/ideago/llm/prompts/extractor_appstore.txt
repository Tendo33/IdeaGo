## Instructions
1. For each result that represents a real iOS app relevant to the user's intent, extract competitor information.
2. Prioritize facts from `appstore_meta` (genre, rating, rating_count, price, version, release_date_iso). Treat `description` only as supporting context.
3. Do not copy marketing text verbatim into `one_liner`.
4. **CRITICAL: Only use URLs that appear in the provided data. NEVER fabricate or guess URLs.**
5. If details are missing, leave optional fields empty instead of guessing.
6. Assign relevance_score (0.0-1.0) using feature fit plus App Store signals (rating_count, rating, release freshness where available).
7. Also extract:
   - `pain_signals` (review pain clusters, severity/frequency)
   - `commercial_signals` (paid intent, conversion clues)
   - `migration_signals` (switch intent away from existing apps)
   - `evidence_items` with explicit categories
8. `evidence_items[].category` must be one of:
   - `competitor`, `pain`, `commercial`, `migration`, `whitespace`, `market`
9. Write all natural-language fields in the specified `output_language`.
10. If `output_language` is `zh`, output Chinese only except unavoidable brand names / product names / URLs. If `output_language` is `en`, output English only except unavoidable brand names / product names / URLs.

## 3. Confirmed Issues

### [P1] 报告状态刷新失败时错误触发 SSE 重试

- Location: [`frontend/src/features/reports/components/useReportStatusResolution.ts:271`](/Users/simonsun/github_project/IdeaGo/frontend/src/features/reports/components/useReportStatusResolution.ts:271)
- Impact: 用户在真实的状态接口失败场景下会继续看到“重连流”动作，错误被伪装成处理中，增加误判和重复操作。
- Evidence: `refreshCurrentReport()` 原来在 `.catch()` 分支里仍然调用 `options?.onProcessing?.()`；而 `retryErrorState()` 会把它绑定到 `retrySSE`。
- Status: 已在本次修复。现在只有在明确拿到 `processing` 状态时才会触发 SSE 重试。

### [P2] 受限浏览器环境下历史缓存可能打断会话清理

- Location: [`frontend/src/features/history/historyCache.ts:51`](/Users/simonsun/github_project/IdeaGo/frontend/src/features/history/historyCache.ts:51)
- Impact: Safari 隐私模式、受限 WebView、禁用存储环境里，`sessionStorage.setItem/removeItem` 抛错会影响 401 登出恢复、历史缓存刷新和显式登出动作。
- Evidence: `writeHistoryCache()` 和 `clearHistoryCache()` 原先直接访问 `window.sessionStorage`，无异常保护；而它们被 `fetchWithTimeout()`、`AuthProvider`、历史页缓存逻辑复用。
- Status: 已在本次修复。现在读写失败会静默降级，不阻断主流程。

### [P2] 管理台额度输入框会保留旧值并覆盖新配额

- Location: [`frontend/src/features/admin/AdminPage.tsx:62`](/Users/simonsun/github_project/IdeaGo/frontend/src/features/admin/AdminPage.tsx:62)
- Impact: 管理员在列表刷新后仍可能提交过期的 `plan_limit`，把后台或其他操作已经更新的额度再次覆盖掉。
- Evidence: `UserRow` 的 `limit` 只在首次渲染从 `user.plan_limit` 初始化，后续 prop 变化不回流到本地状态。
- Status: 已在本次修复。现在非编辑态会同步最新 `plan_limit`。

## 4. Likely Risks

### [P2] 多实例下管理台指标与运行态视图不完整

- Location: [`src/ideago/api/runtime_state.py:14`](/Users/simonsun/github_project/IdeaGo/src/ideago/api/runtime_state.py:14), [`src/ideago/api/routes/admin.py:194`](/Users/simonsun/github_project/IdeaGo/src/ideago/api/routes/admin.py:194)
- Impact: 一旦 `saas` 部署到多进程或多实例，`/admin/metrics` 只会返回单实例请求快照，SSE 运行态、pipeline task 和 processing registry 也只在本进程可见，排障时容易得出错误结论。
- Evidence: `ReportRunRegistry`、`ProcessingDedupRegistry`、`PipelineTaskRegistry` 都是进程内对象；`admin_metrics()` 直接返回 `app_metrics.snapshot()`。
- Recommendation: 把 admin metrics 对齐到集中式指标源，把至少 processing/report-run 可见性接到共享状态或明示为“当前实例视角”。

### [P2] 回归验证成本过高，影响快速修复效率

- Location: [`pyproject.toml:187`](/Users/simonsun/github_project/IdeaGo/pyproject.toml:187), [`tests/test_api.py`](/Users/simonsun/github_project/IdeaGo/tests/test_api.py), [`tests/test_langgraph_engine.py`](/Users/simonsun/github_project/IdeaGo/tests/test_langgraph_engine.py)
- Impact: 小范围修复也容易被全局 coverage 门槛和超大测试文件拖慢，自动化线程更难做“最小验证后立即反馈”。
- Evidence: `pytest` 全局开启 `--cov-fail-under=80`；同时测试集中在 `tests/test_api.py` 5293 行、`tests/test_langgraph_engine.py` 3089 行。
- Recommendation: 增加小粒度 smoke/contract 测试入口，或为定向回归提供不走全局 coverage 的本地命令。

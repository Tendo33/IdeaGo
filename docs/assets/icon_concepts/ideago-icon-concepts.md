# IdeaGo Icon Concepts & Prompt Pack

## 1. Repository Reading
**Parsed files:** `README.md`, `pyproject.toml`, `frontend/package.json`
**Key structures:**
- **Backend pipeline:** LangGraph engine handling intent parsing, query planning, platform adaptation, extraction, and aggregation.
- **Six core sources:** Tavily, Reddit, GitHub, Hacker News, App Store, Product Hunt.
- **Frontend:** React workspace with real-time SSE progress streaming and structured validation reports.

## 2. Project Essence
**Product Name:** IdeaGo  
**Product Category:** AI-powered competitor research & idea validation engine.  
**Core Capabilities:** Real-time multi-source data extraction, signal aggregation, decision-first reporting.  
**Interaction Feel:** Structured, pipeline-driven, fast, transparent.  
**Brand Temperament:** Analytical, authoritative, precise, calm, and intelligent.  
**Anti-directions:** Literal brains or lightbulbs, disorganized data clouds, playful or casual startup blobs.

## 3. Style Mapping
The core identity translates best to:
- **Workflow / Pipeline systems:** Representing the multi-step aggregation of evidence from various sources.
- **AI Workbench / Hub:** Representing the central point where external chaos becomes structured insight.
- **Shape primitives:** Dots (data/sources), simple paths/lines (pipelines), central solid forms (the final report/validation).

## 4. Concept A: The Aggregator Core
*Closest to the repository structure*
- **Metaphor:** Six external signals converging into a single unified truth.
- **Geometry:** A central solid rounded square with six short, thin strokes or small dots trailing out from its edges.
- **Why it fits the repository:** Directly represents the pipeline's six live sources feeding the central LangGraph orchestrator.
- **Why it fits the reference style:** Uses primitive geometry (square + strokes) and is highly structural and symmetrical.
- **Primary risk:** May feel too much like a generic "API hub" or network node if not kept minimalist.

## 5. Concept B: The Synthesis Matrix
*Closest to the reference style language*
- **Metaphor:** Noise being analyzed and focused into pure signal.
- **Geometry:** A grid or matrix of tiny dots where the outer edges are sparse and dim, coalescing into a dense, bright central core.
- **Why it fits the repository:** Reflects the core "idea validation" process—distilling vast web data into a single authoritative decision.
- **Why it fits the reference style:** Highly parametric, relies on repeating primitives (dots), calm and generative.
- **Primary risk:** If the dot variation is too random, it will look like a starry sky instead of a purposeful data matrix.

## 6. Concept C: The Validation Lens
*Strongest standalone brand symbol*
- **Metaphor:** Precision focus and finding the opportunity intersection.
- **Geometry:** Two overlapping, delicate minimalist circles (the "ideas and market") with a single bold, solid white dot perfectly placed at their intersection.
- **Why it fits the repository:** Communicates finding the exact "whitespace opportunity" or truth within intersecting markets and signals.
- **Why it fits the reference style:** Extremely scalable, vector-friendly, pure minimalist geometry with massive negative space.
- **Primary risk:** Might appear too simplistic if the atmospheric background doesn't provide enough premium context.
